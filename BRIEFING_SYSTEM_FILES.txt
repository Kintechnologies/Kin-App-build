4. apps/web/src/app/api/morning-briefing/route.ts
   - GET: Returns today's briefing (or generates new)
     * Queries calendar, schedule, kids, budget, pet care
     * Uses Claude to generate warm briefing
     * Stores in morning_briefings table
     * Returns: { content, deliveryStatus }
   
   - POST: Forces regeneration of today's briefing
     * Deletes existing, generates new
     * Returns: { content, deliveryStatus }

5. apps/web/src/app/api/push-tokens/route.ts
   - GET: Returns all active push tokens for user
     * Returns: { tokens }
   
   - POST: Saves a push token
     * Request: { token, platform, device_name? }
     * Upserts to push_tokens table
     * Returns: { success, token_id }
   
   - DELETE: Deactivates a push token
     * Request: { token }
     * Soft delete: sets active = false
     * Returns: { success }

6. supabase/functions/morning-briefing/index.ts
   - Deno/TypeScript edge function
   - Runs daily at 6am to generate and send briefings
   - Process:
     * Gets all profiles with onboarding_completed = true
     * For each: generates briefing, gets push tokens, sends notification
     * Stores in morning_briefings table
     * Handles errors gracefully per-user
   - Environment variables:
     * SUPABASE_URL
     * SUPABASE_SERVICE_ROLE_KEY
     * ANTHROPIC_API_KEY
     * EXPO_ACCESS_TOKEN (optional)
   - Response: { processed, succeeded, failed, errors }

==========================================================================
KEY FEATURES
==========================================================================

SAFETY & PRIVACY
- Allergy context ALWAYS included (non-negotiable)
- Fitness data strictly private per parent
- Budget shows only totals, not line items
- Calendar respects shared/private event boundaries

VOICE & TONE
- No hedging ("I think", "perhaps")
- Always specific numbers
- One question maximum per briefing
- Warm, direct, human language
- No corporate jargon
- 30-60 second read time

INTEGRATION POINTS
- Mobile app registers for push on startup
- Web API generates briefings on-demand or scheduled
- Supabase edge function sends daily at 6am
- Claude used for warm, natural briefing generation
- Expo used for push delivery

==========================================================================
DEPLOYMENT STEPS
==========================================================================

1. Apply migrations 013-018 to Supabase
2. Set environment variables in edge function:
   - ANTHROPIC_API_KEY
   - EXPO_ACCESS_TOKEN
3. Deploy edge function and set schedule (6am UTC)
4. Deploy web API routes
5. Update mobile app with new libraries
6. Initialize push notifications on app startup
7. Test end-to-end: register token → generate briefing → verify storage
