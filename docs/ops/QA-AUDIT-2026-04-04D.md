## Prompt Drift — NEW P2 FOUND

### ⚠️ P2-11 (Lead Eng): `morning-briefing/route.ts` inline prompt missing relief language selection guide

The inline system prompt in `apps/web/src/app/api/morning-briefing/route.ts` (lines ~360–364) lists the three relief phrases with short category labels only:

```
- Time-based: "I'll remind you when it's time to leave."
- Monitoring: "I'll keep an eye on it."
- Conditional: "I'll flag it if anything changes."
```

The canonical `docs/prompts/morning-briefing-prompt.md` includes decision guidance for each:
```
- "I'll remind you when it's time to leave." → use when there is a specific departure or action time
- "I'll keep an eye on it." → use when an unresolved issue exists but is not yet escalated
- "I'll flag it if anything changes." → use when the current state is adequate but dynamic; Kin is in standby
```
Plus: `"One relief line max per briefing. Only include if monitoring is genuinely warranted."`

This guidance was dropped when the prompt was inlined in the route. Relief phrase mismatch is explicitly listed as Known Failure Mode #5 in the prompt file. Without selection guidance, the model may choose the wrong phrase in ambiguous situations (e.g., using "I'll keep an eye on it" when a hard departure time is known).

**Fix:** Add the selection guide lines to the inline system prompt in `route.ts`, matching the prompt file verbatim.

**Priority:** P2 — does not block TestFlight but risks user-facing tone inconsistency.

**Owner:** Lead Eng
