### P2-NEW-2 — `CHAT_SYSTEM_PROMPT` `## CONTEXT PROVIDED PER MESSAGE` section does not document `partner_today_events` or `partner_recent_schedule_changes`

**File:** `apps/web/src/app/api/chat/route.ts`, lines 137–142

**Issue:**
The `CHAT_SYSTEM_PROMPT` (used as fallback for household threads while `HOUSEHOLD_CHAT_SYSTEM_PROMPT` is a placeholder) documents 4 context keys at the end of the prompt:

```
## CONTEXT PROVIDED PER MESSAGE
Each message will include:
- speaking_to: "parent_a" | "parent_b"
- today_events: array of household events
- open_coordination_issues: array of OPEN/ACKNOWLEDGED items from coordination_issues table
- recent_schedule_changes: events updated in the last 24 hours
```

For household threads, the actual context JSON now includes two additional keys:
- `partner_today_events` — partner's calendar events for today (added Run AN)
- `partner_recent_schedule_changes` — partner's calendar changes in last 24h (added Run AR)

These keys are present in the JSON block sent to Claude but are not described in the active system prompt. The model infers their meaning from the key names, which is pragmatically workable, but is not spec-compliant.

**Current runtime risk:** Low. `HOUSEHOLD_CHAT_SYSTEM_PROMPT` is still an empty placeholder — the personal thread prompt is active for household threads in both dev and production. The model likely handles undocumented JSON keys correctly. No user-facing failure observed.

**Risk at household prompt delivery:** When IE delivers `household-chat-prompt.md` and Lead Eng wires it, the new system prompt MUST document both partner context keys in its `## CONTEXT PROVIDED PER MESSAGE` section. If IE is unaware of these keys, the household prompt may be authored without them, degrading §16 social tone output (model may not use partner schedule data to assign balanced responsibility).

**Action required (Intelligence Engineer + Lead Eng):** Ensure `household-chat-prompt.md` when authored explicitly documents `partner_today_events` and `partner_recent_schedule_changes` in its context section. Lead Eng to include a comment in `route.ts` listing all active context keys as a reference for IE.

**Severity:** P2 — does not block TestFlight; must be addressed before `household-chat-prompt.md` is wired.
