### P2-NEW — settings.tsx: Calendar badge uses rose semantic colors instead of neutral

**File:** `apps/mobile/app/(tabs)/settings.tsx` styles `badge` / `badgeText` (lines 534–547)

**Spec (`settings-screen-spec.md` §8 Badge pill):**
| Property | Spec | Actual |
|----------|------|--------|
| Badge background | `c.surfaceSubtle` | `c.roseSubtle` |
| Text color | `c.textFaint` | `c.rose` |
| Border radius | 20px | 8px |
| Padding horizontal | 4px | 10px |
| Padding vertical | 6px | 4px |

The "Not connected" badge renders with rose (pink-red) colors. Rose is the semantic color for family members / alerts — it is the wrong semantic register for a neutral connection status. A user seeing a pink-red "Not connected" badge may interpret it as an error or warning state. The spec calls for neutral muted styling (`c.surfaceSubtle` / `c.textFaint`).

**Action required (Lead Eng):** Update `badge.backgroundColor` to `c.surfaceSubtle`, `badgeText.color` to `c.textFaint`, `badgeText.letterSpacing` to `0.5` (currently `0.5` — ✅ matches), `badge.borderRadius` to 20, and `badge.paddingHorizontal/paddingVertical` to 4/6.
**Severity:** P2 — visual semantic error (misleading color)

### P2-NEW — settings.tsx: Sign out fires haptic on destructive action

**File:** `apps/mobile/app/(tabs)/settings.tsx` `handleSignOut` function (line 83)

**Code:**
```ts
onPress: () => {
  Haptics.impactAsync(Haptics.ImpactFeedbackStyle.Medium);
  signOut();
},
```

**Spec (`settings-screen-spec.md` §12 Sign Out Button):**
> Haptic: Not applicable (destructive action, no haptic reinforcement)

The spec explicitly disallows haptic feedback for sign out. A Medium impact haptic on a destructive action provides positive reinforcement for an irreversible step — the opposite of the intended UX.

**Action required (Lead Eng):** Remove `Haptics.impactAsync(Haptics.ImpactFeedbackStyle.Medium)` from the sign-out `onPress` handler.
**Severity:** P2
