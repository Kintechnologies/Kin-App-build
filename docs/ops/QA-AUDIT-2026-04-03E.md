## Issues Found

### P0 — NONE

### P1 — 1 issue

**P1-1 — `morning-briefing/route.ts` system prompt missing §5 sentence cap**
- **File:** `apps/web/src/app/api/morning-briefing/route.ts` — system prompt, lines 327–344
- **Spec:** §5 — "never more than 4 sentences" for morning briefing
- **Finding:** System prompt instructs "Short by default - readable in 30-60 seconds" but does not specify a 4-sentence maximum. The client-side `parseBriefingBeats(...).slice(0, 4)` truncates display correctly, but:
  1. The AI may generate logical output in sentence 5+ that is silently dropped on the client, confusing the narrative
  2. The spec requires the AI not to generate output exceeding the limit — client truncation is a workaround, not compliance
- **Fix:** Add to system prompt: `"Your entire briefing must be 4 sentences or fewer. No exceptions. If you have more to say, keep only the highest-priority items."`
- **Note:** This file was modified in Run I (B20). The system prompt content was not the focus of B20 but is in-scope for this audit. This issue was not flagged in prior QA runs.

### P2 — 1 issue

**P2-1 — `fitness.tsx` unused imports: `TouchableWithoutFeedback`, `FlatList`**
- **File:** `apps/mobile/app/(tabs)/fitness.tsx` lines 14, 16
- **Finding:** Both are imported from `react-native` but have no JSX usage (`<TouchableWithoutFeedback` and `<FlatList` do not appear in the file)
- **Context:** B6 fixed console.error gating in `fitness.tsx` but did not clean unused imports. B5 cleaned equivalent imports from `family.tsx` but `fitness.tsx` was not included in that cleanup.
- **Fix:** Remove `TouchableWithoutFeedback` and `FlatList` from the import statement
- **Impact:** Cosmetic / tree-shaking; not user-visible. `fitness.tsx` is a data-layer file not in navigation.
