## P0 Finding — COMMITTED REGRESSION

### P0-NEW-BH-1 — InstrumentSerif-Italic removed from font registration; 6 usages remain in HEAD

**Severity:** P0 — COMMITTED runtime regression. Blocks TestFlight. Blocks App Store screenshots.

**Commit:** `68c270e` — "fix: font registration and Geist-SemiBold wiring"

**What the commit did:**
Removed `"InstrumentSerif-Italic": require("../assets/fonts/InstrumentSerif-Italic.ttf")` from `useFonts()` in `apps/mobile/app/_layout.tsx`.

**What was NOT updated:**
`index.tsx` and `chat.tsx` still reference `InstrumentSerif-Italic` in 6 style definitions that remain in HEAD:

| File | Line | Style | Element |
|------|------|-------|---------|
| `apps/mobile/app/(tabs)/index.tsx` | 885 | `displayName` | Today header — user's name (first element seen) |
| `apps/mobile/app/(tabs)/index.tsx` | 922 | `briefingTitle` | "Morning" label on briefing card |
| `apps/mobile/app/(tabs)/index.tsx` | 949 | `briefingHook` | Briefing hero sentence (Screenshot 1/2 hero) |
| `apps/mobile/app/(tabs)/index.tsx` | 1133 | `cleanDayText` | "Clean day." silence state text |
| `apps/mobile/app/(tabs)/chat.tsx` | 937 | `listTitle` | "Conversations" screen title |
| `apps/mobile/app/(tabs)/chat.tsx` | 985 | `pinnedThreadName` | Pinned thread name (Kin, Household) |

**Runtime behavior:** React Native silently falls back to the system default font when a requested font is not registered. All 6 elements render in system font on TestFlight builds. No crash — but the visual spec is broken silently.

**Impact:**
- Briefing card hero (`briefingHook`) loses its Instrument Serif voice — the deliberate emotional/personal register Kin is designed around
- User's display name loses its warm-serif welcome treatment
- Silence state loses its spec-required low-weight italic rendering
- Conversations list title and pinned thread name lose spec typography
- App Store screenshots are blocked until resolved (screenshots spec production checklist requires "Typography rendering correctly: Instrument Serif Italic loading")
- P2-TYPO-1/2/3/4 resolutions from Run BD are silently undone at runtime

**History of this issue:** This was P1-NEW-BE-1 in Runs BE/BF/BG — flagged as a working-tree risk that must NOT be committed. Commit `68c270e` committed the broken state without resolving the usage gap.

**Required fix — two valid paths:**

**Option A (preferred — spec-preserving):** Restore InstrumentSerif-Italic registration to `_layout.tsx`:
```tsx
"InstrumentSerif-Italic": require("../assets/fonts/InstrumentSerif-Italic.ttf"),
```
Add back the one line that was removed. No changes to index.tsx or chat.tsx needed.

**Option B (design change — requires P&D sign-off):** Replace all 6 InstrumentSerif-Italic references in `index.tsx` and `chat.tsx` with a registered Geist variant. This constitutes a design change to all hero elements and requires P&D review before implementing.

**Recommended path:** Option A. Restore font registration. This is a one-line change.
