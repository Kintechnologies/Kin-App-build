# Quick Start: Kin Onboarding Flow

**For:** Developers integrating Track B into the app
**Time to read:** 5 minutes

---

## What You Just Got

A complete 5-screen onboarding flow that:
- Collects family data in 3 minutes
- Generates a real morning briefing as proof
- Persists everything to Supabase
- Hands a ready-to-use household context to Track D

---

## 30-Second Install

```typescript
// In your app's root navigator
import { OnboardingNavigator } from './engineering/screens/onboarding/onboarding-navigator'

export function App() {
  const [userId, setUserId] = useState<string | null>(null)

  if (!userId) {
    return (
      <OnboardingNavigator
        onComplete={(userId, parentId, householdId) => {
          setUserId(userId)
          // You're done — navigate to Track D's Today tab
        }}
        onError={(error) => {
          console.error(error)
          // Show error UI
        }}
      />
    )
  }

  return <YourMainApp userId={userId} />
}
```

Done. That's the integration.

## What Happens

1. **Screen 1** — User enters family name + location (10 sec)
2. **Screen 2** — User describes typical weekday (60 sec)
   - Claude parses into schedule pattern
   - User confirms summary
3. **Screen 3** — User adds children/pets with allergies (90 sec)
4. **Screen 4** — User enters monthly grocery budget (15 sec)
5. **Screen 5** — Shows morning briefing preview
   - User creates account
   - Allergies saved to Supabase
   - All data persisted
   - `onComplete()` fires

---

## Key Facts

- **Allergy handling:** Multi-select. A child can have dairy AND egg allergies.
- **FVM briefing:** Real, specific, generated by Claude from user's data.
- **Account creation:** Deferred to after briefing (Apple HIG best practice).
- **Database writes:** Only at end of Screen 5 (atomic save).
- **Error handling:** Graceful degradation — logs errors but doesn't crash.

## After Onboarding Completes

Track D can expect:

**In Supabase:**
- `households` — family_name
- `parents` — home_location, timezone, briefing_time, onboarding_complete=true
- `children` — name, age, grade
- `children_allergies` — allergen per child (separate table)
- `pets` — name, species, breed, vet_name
- `budget_categories` — 6 defaults (Groceries = user's budget)
- `kin_memory` — schedule pattern (memory_type='fact')
- `trial_state` — 7-day free trial

**Query example:**
```typescript
const { data: household } = await supabase
  .from('households')
  .select('*')
  .eq('id', householdId)
  .single()

const { data: children } = await supabase
  .from('children')
  .select('*')
  .eq('household_id', householdId)

const { data: allergies } = await supabase
  .from('children_allergies')
  .select('*')
  .eq('household_id', householdId)
```

## Critical Details

### Allergies
When Track D generates meals, **always check allergies**:
```typescript
const childAllergies = allergies.filter(a => a.child_id === child.id)
const forbiddenFoods = childAllergies.map(a => a.allergen)
// Never suggest anything in forbiddenFoods
```

### Schedule
The schedule pattern is stored in `kin_memory`:
```typescript
const { data: memory } = await supabase
  .from('kin_memory')
  .select('*')
  .eq('household_id', householdId)
  .eq('memory_type', 'fact')

const schedulePattern = memory
  .find(m => m.content?.type === 'schedule_pattern')
  ?.content?.data

// Access: schedulePattern.wake_time, .work_start_time, .school_pickup_time, etc.
```

### Timezone & Briefing Time
```typescript
const { data: parent } = await supabase
  .from('parents')
  .select('*')
  .eq('id', parentId)
  .single()

console.log(parent.timezone) // "America/New_York"
console.log(parent.briefing_time) // "06:00:00"
```

## Troubleshooting

**"Allergies not showing?"**
- Check `children_allergies` table, not `children`
- Filter by `household_id` AND `child_id`

**"Schedule pattern is null?"**
- Look in `kin_memory` (not in children or parent tables)
- Filter by `memory_type = 'fact'` and `content.type = 'schedule_pattern'`

**"Parent record missing fields?"**
- Check `onboarding_complete = true` on parent
- home_location set from Screen 1
- briefing_time set from auth (defaults: 06:00 for Partner 1)
